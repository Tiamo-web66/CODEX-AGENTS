# AGENTS.md

本文件用于说明 Codex/Agent 在本仓库中的默认工作方式。若用户的明确指令与本文冲突，以用户当前指令为准。

## 工作原则

- 先理解当前仓库结构和已有文件，再修改。
- 保持变更聚焦，避免无关重构和批量格式化。
- 不回滚用户已有修改，除非用户明确要求。
- 对会影响历史、远端或大量文件的操作，先说明风险。
- 优先使用仓库现有约定；没有约定时，采用简单、清晰、可维护的实现。

## 文件组织

- `README.md`：面向使用者的仓库介绍和入口说明。
- `AGENTS.md`：Agent 协作规范和本仓库维护规则。
- `docs/`：较长的说明文档、设计记录、流程规范。
- `prompts/`：可复用提示词。
- `scripts/`：辅助脚本。
- `templates/`：可复用模板。

## 编辑规范

- 默认使用 UTF-8。
- Markdown 文档保持标题层级清晰，避免堆砌空洞说明。
- 脚本应尽量可重复运行，并在文件内或相邻文档中说明运行方式。
- 新增目录前确认它会被实际使用；不要为了占位创建复杂结构。

## 验证习惯

- 文档类修改至少检查拼写、链接和目录结构。
- 脚本类修改应运行对应命令或提供无法运行的原因。
- 涉及 Git 操作时，完成后检查 `git status --short --branch`。
